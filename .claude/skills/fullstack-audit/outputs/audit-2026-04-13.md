## Architecture & Code Quality

### Medium

- [x] **#1 `UsersService` creates its own `CognitoIdentityProviderClient` bypassing typed config** ✅ FIXED — Mirrored the `AuthService` pattern: reads `aws.region`, `aws.credentials.accessKeyId`, and `aws.credentials.secretAccessKey` from `ConfigService` using typed paths; credentials spread conditionally so IAM role fallback is preserved; throws at startup if `aws.region` is missing. Found in: `apps/api/src/users/users.service.ts:27-29`

- [ ] **#2 Three remaining `catch (error: any)` annotations in production service code** ❌ OPEN — `S3Service.keyExists()`, `PushNotificationService.createEndpoint()`, and `PushNotificationService.publishToTokens()` all use `catch (error: any)`. A shared AWS SDK error type or type predicate would preserve property access without losing type safety. Found in: `apps/api/src/common/s3.service.ts:257`, `apps/api/src/common/push-notification.service.ts:59,179`

### Low

- [ ] **#3 `ChatGateway` reads `process.env` directly at module-load time for CORS origins** ❌ OPEN — Due to decorator evaluation timing, the WebSocket CORS config uses raw `process.env` rather than validated `ConfigService`. Found in: `apps/api/src/chat/chat.gateway.ts:28-35`

- [ ] **#4 `PrismaExceptionFilter.catch()` accesses HTTP response without verifying context type** ❌ OPEN — If a Prisma error propagated from a WebSocket handler, `switchToHttp()` would cause a secondary exception. Found in: `apps/api/src/prisma/prisma-exception.filter.ts:9`

## Security

### High

- [x] **#5 JWT `issuer` and `audience` claims are not validated in either the REST or WebSocket auth paths** ✅ FIXED — Added `issuer` and `audience` validation to both the passport-jwt `JwtStrategy` (REST) and `ChatGateway.verifyToken()` (WebSocket). Tokens from other User Pools or intended for other app clients are now rejected. Also migrated `JwtStrategy` from raw env var names (`AWS_REGION`, `COGNITO_USER_POOL_ID`) to typed config paths (`aws.region`, `aws.cognito.userPoolId`, `aws.cognito.clientId`). Found in: `apps/api/src/auth/strategies/jwt.strategy.ts`, `apps/api/src/chat/chat.gateway.ts`

### Medium

- [ ] **#6 WebSocket `sendMessage` handler has no per-connection rate limiting** ❌ OPEN — The `ThrottlerModule` only applies to HTTP routes. A single authenticated WebSocket client could flood a conversation with messages at connection speed, generating excessive DB writes and push notifications. Found in: `apps/api/src/chat/chat.gateway.ts:163-204`

- [ ] **#7 CSP `img-src` directive allows `https:` (any HTTPS origin)** ❌ OPEN — `img-src 'self' data: blob: https:` is broader than necessary. An XSS payload could exfiltrate data via image src to any HTTPS endpoint. Should be restricted to known origins (CloudFront, S3, Google Maps, googleusercontent.com). Found in: `apps/web/middleware.ts:72`

- [ ] **#8 `SentReminder` model has no foreign key constraints on `eventId` or `userId`** ❌ OPEN — When events/users are deleted, `SentReminder` rows become orphaned with no cascade cleanup. Found in: `apps/api/prisma/schema.prisma:277-284`

### Low

- [ ] **#9 `isSameOriginRequest` returns `true` when no `Origin` or `Sec-Fetch-Site` header is present** ❌ OPEN — Non-browser clients (curl) bypass the CSRF check. Practical risk is very low since the routes require valid auth tokens. Found in: `apps/web/lib/server-request.ts:14`

- [ ] **#10 `ChatGateway.handleConnection` does not emit an error event before disconnecting unauthenticated clients** ❌ OPEN — Clients cannot distinguish "server rejected my token" from "network error." Found in: `apps/api/src/chat/chat.gateway.ts:89-133`

## Performance

### Medium

- [ ] **#11 `ChatService.sendMessage()` creates in-app notifications per-recipient in serial fire-and-forget** ❌ OPEN — Uses individual `notification.create()` and `sendToUser()` calls instead of batch `createMany()` and `sendToUsers()`. For group chats with many participants, this generates N separate SQL INSERTs and N separate push calls. Found in: `apps/api/src/chat/chat.service.ts:225-253`

- [ ] **#12 `BookmarksService.findAll()` does not strip `imageKey` from the response** ❌ OPEN — Leaks internal S3 keys in API responses, unlike other services that consistently strip `imageKey`/`avatarKey`/`coverImageKey`. Found in: `apps/api/src/bookmarks/bookmarks.service.ts:58-103`

### Low

- [ ] **#13 `StatsService.getHomeStats()` computes week bounds using server-local timezone** ❌ OPEN — Uses `setHours(0, 0, 0, 0)` which operates in server-local time. For a Ukrainian platform, the "this week" count may be off at the boundary. Found in: `apps/api/src/stats/stats.service.ts:65-74`

## Database & Prisma

### Medium

- [ ] **#14 `SentReminder` model lacks foreign key relations, preventing cascade cleanup** ❌ OPEN — No referential integrity, orphaned rows grow unbounded, and the model only has the `@@unique([eventId, userId])` index. Found in: `apps/api/prisma/schema.prisma:277-284`

### Low

- [ ] **#15 Duplicate notification delivery for followers who are also geographically nearby** ❌ OPEN — A user who is both a follower AND within the nearby radius receives two notifications for the same event (different types). Found in: `apps/api/src/events/events.service.ts:660-700+`

- [ ] **#16 `Conversation.directKey` uses multiple formats without documented constraint** ❌ OPEN — Stores `${userA}:${userB}` for DMs, `event:${eventId}` for event chats, `group:${groupId}` for group chats, with no validation ensuring format correctness. Found in: `apps/api/src/chat/chat.service.ts:109-111`, `apps/api/src/events/events.service.ts:574`

## AWS Integration

### Medium

- [ ] **#17 ECS tasks still run in public subnets with `assignPublicIp=ENABLED`** ❌ OPEN — Remaining from previous audit #31. Needs VPC endpoints (~$35/mo) or NAT gateway (~$32/mo per AZ) to move to private subnets. Found in: `.github/workflows/deploy.yml:104-134`, `infra/NETWORKING.md`

- [ ] **#18 Legacy development RDS remains publicly accessible with 0.0.0.0/0 ingress** ❌ OPEN — `kroogom-db` in default VPC has `PubliclyAccessible=true` and may contain PII from the production migration snapshot. Password should be rotated and access restricted. Found in: `infra/NETWORKING.md:69-80`

### Low

- [ ] **#19 No S3 orphaned image reconciliation** ❌ OPEN — Failed deletions and unused uploads leave orphaned S3 objects. Only `AbortIncompleteMultipartUploads` lifecycle rule exists. Found in: `infra/s3-lifecycle.json`

- [ ] **#21 No `HEALTHCHECK` instruction in the API Dockerfile** ❌ OPEN — The web Dockerfile has one; the API Dockerfile does not. Found in: `apps/api/Dockerfile`

- [ ] **#22 Docker image digests are 18 days old with no automated refresh** ❌ OPEN — Pinned to 2026-03-26 digest. No Dependabot/Renovate config for automated digest updates. Found in: `apps/api/Dockerfile:3,23`, `apps/web/Dockerfile:3,32`
