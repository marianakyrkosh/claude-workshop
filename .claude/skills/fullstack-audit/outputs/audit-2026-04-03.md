# Production Audit Report

**Project**: Kroogom
**Audited**: 2026-04-03
**Stack**: Next.js 15 + NestJS 10 + Prisma + PostgreSQL + AWS (ECS Fargate, Cognito, S3, SNS)

## Summary

Kroogom is a well-structured monorepo with solid foundational security (Helmet, ValidationPipe with `whitelist`/`forbidNonWhitelisted`, JWKS-based JWT validation, magic-number file validation) and a mature CI/CD pipeline using OIDC for AWS credentials. The primary risks are a TOCTOU race condition on event capacity enforcement that can allow overbooking under concurrent load, tokens stored in `sessionStorage` instead of HTTP-only cookies which exposes them to XSS, and a set of hardcoded production infrastructure identifiers committed to the deploy workflow. Performance and database findings are largely medium-severity with clear paths to resolution.

## Architecture & Code Quality

### High

- [x] **#1 Auth service issues Cognito AdminGetUser API calls on every token validation** âś… FIXED (PR #242) â€” Added a 5-minute in-process TTL cache (`Map<sub, {value, expiresAt}>`) in `AuthService`. `enrichPayloadFromCognito` checks the cache before calling `AdminGetUserCommand`; cache hits skip the Cognito round-trip entirely. Null results cached too. Unit tests cover hit, miss, cross-user isolation, expiry, and singleflight/concurrent coalescing.

### Medium

- [x] **#2 Application-level distance sort operates on paginated data, not the full result set** âś… FIXED â€” When `sortBy=distance`, `searchAdvanced` now fetches the full bounding-box-filtered result set without `skip`/`take`, sorts all events by Haversine distance, then slices to the requested page. Cross-page ordering is now coherent. Two new tests cover within-page ordering and cross-page ordering. Non-distance sort paths are unaffected.

- [x] **#3 Repeated per-request S3 signed URL generation for every list response** âś… N/A â€” Superseded by CloudFront/CDN migration (2026-04-04). `S3Service.getImageUrl()` now returns a CDN URL synchronously when `cdnDomain` is configured; `getSignedUrl` is no longer called on list responses. No presign calls are generated.

- [x] **#4 `DIRECT_DATABASE_URL` is not validated in `env.validation.ts`** âś… FIXED â€” Added `@IsString() @IsOptional() DIRECT_DATABASE_URL?: string` to `EnvironmentVariables`. Added a production guard in `validate()` that throws a clear error if `NODE_ENV === production` and the variable is absent. Development and test environments remain unaffected. Five new tests added to `env.validation.spec.ts`; all 676 tests pass.

### Low

- [x] **#5 `any` type used in hot code paths in `events.service.ts`** âś… FIXED â€” Removed three explicit `: any` annotations from `findNearby`'s Haversine map/filter/sort callbacks. TypeScript now infers the element type from `.map()`'s return shape (spread of paginated event with `distance: number` added); no `any` remains in the hot path.

- [x] **#6 Tag update in `events.service.ts` uses two separate database operations without a transaction** âś… FIXED â€” Wrapped the disconnect-then-reconnect tag mutation in `$transaction`. The `set: []` disconnect and the `connectOrCreate` reconnect now execute atomically; a failure in the second update rolls back the first, preventing the event from being left with no tags. One new atomicity test added.

## Security

### Critical

- [x] **#7 Auth tokens stored in `sessionStorage`, not HTTP-only cookies** âś… FIXED (PR #240) â€” Refresh token moved to HTTP-only SameSite=Lax cookie via `GET/POST/DELETE /api/auth/session` route. Access/ID tokens are memory-only (React state). Session restore on reload calls `GET /api/auth/session` server-side. PKCE state/verifier remain in sessionStorage (short-lived, non-secret).

### High

- [x] **#8 Production Cognito User Pool ID and Client ID hardcoded in the deploy workflow** âś… FIXED â€” Cognito User Pool ID and Client ID moved to SSM (`/kroogom/prod/cognito-user-pool-id`, `/kroogom/prod/cognito-web-client-id`) and fetched at runtime in the existing "Pull secrets from SSM" step. `--build-arg` values now reference shell variables. ECR registry sourced from `amazon-ecr-login@v2` action output. IAM role ARN moved to `${{ secrets.AWS_DEPLOY_ROLE_ARN }}` GitHub environment secret. No hardcoded AWS identifiers remain in the workflow. Also fixes #15.

- [x] **#9 `unsafe-inline` on `script-src` in production CSP completely negates XSS protection** âś… FIXED â€” CSP moved from static `next.config.js` headers to middleware. A 128-bit cryptographically random nonce is generated per request via `crypto.getRandomValues`. Production `script-src` now uses `'nonce-{nonce}' 'strict-dynamic'` â€” `unsafe-inline` removed entirely. Development adds `'unsafe-eval'` for HMR. Nonce forwarded as `x-nonce` request header; Next.js stamps it on hydration scripts automatically. `GoogleAnalytics` updated to accept and propagate `nonce` prop. Fixed in PR #249.

- [x] **#10 Global throttle rate (120 req/min) allows credential stuffing against the Cognito token exchange endpoint** âś… FIXED â€” Added `THROTTLE_AUTH = { default: { limit: 10, ttl: 60_000 } }` constant and applied `@Throttle(THROTTLE_AUTH)` to `GET /v1/users/cognito/:cognitoId`, `POST /v1/device-tokens`, and `DELETE /v1/device-tokens`. These auth-adjacent endpoints are now limited to 10 req/min per IP, down from the global 120.

- [x] **#11 `findAndRelinkByEmail` unconditionally re-links `cognitoId` based on email alone** âś… FIXED â€” Added `email_verified?: boolean` to `JwtPayload`; `findAndRelinkByEmail` now returns `null` with a warning if `email_verified !== true`, blocking the relink before any DB query. Verified users (native confirmed + all Google-federated) are unaffected. Unit tests cover verified path (relinks), unverified path (blocked), and missing-claim path (blocked).

### Medium

- [x] **#12 `remotePatterns` in `next.config.js` allows any `amazonaws.com` subdomain for image optimization** âś… FIXED â€” The image `remotePatterns` configuration uses `hostname: '**.amazonaws.com'` (wildcard subdomain), which permits Next.js image optimization to proxy and cache images from any AWS-hosted URL, not just the application's S3 bucket. This could be abused to cause the Next.js server to fetch and cache arbitrary content from any public S3 bucket or other AWS endpoint on behalf of users. Found in: `apps/web/next.config.js:29`

- [x] **#13 No rate limiting on Cognito direct API calls from the web client** âś… FIXED (PR #262) â€” Six sensitive actions (sign-in, sign-up, confirm-sign-up, resend-code, forgot-password, confirm-forgot-password) are now proxied through `POST /api/auth/cognito`, which applies a per-IP sliding-window rate limit (10 req/60s) before forwarding to Cognito. `auth-context.tsx` updated to call the proxy. OAuth/PKCE and globalSignOut paths unchanged. `RateLimiter` class includes periodic eviction to prevent unbounded Map growth under bot traffic.

### Low

- [x] **#14 Pre-signup Lambda uses `console.log` instead of structured logging** âś… FIXED â€” Replaced all three `console.log` calls with a `log(level, message, context)` helper that emits JSON to stdout. CloudWatch Logs Insights can now filter on `level`, `outcome`, and `provider` fields. No PII (email, providerUserId, internal username) is written to logs. 7 unit tests added using Node.js built-in `node:test` runner covering all branches including skip, warn, link outcomes, and AdminLinkProviderForUserCommand parameter validation.

- [x] **#15 ECR registry account ID hardcoded in public deploy workflow** âś… FIXED â€” Fixed as part of #8. `ECR_REGISTRY` env var removed; registry URL now sourced from `amazon-ecr-login@v2` `outputs.registry`. IAM role ARN moved to `${{ secrets.AWS_DEPLOY_ROLE_ARN }}`. No AWS account ID remains in the workflow file.

## Performance

### High

- [x] **#16 N+1 avatar-signing in group list responses** âś… N/A â€” Superseded by CloudFront/CDN migration (2026-04-04). `S3Service.getImageUrl()` returns a CDN URL synchronously; no presign calls are made per member. The N+1 cost no longer applies.

### Medium

- [x] **#17 Pre-signed S3 URLs returned in API responses expire after 1 hour but are cached by React Query with no TTL alignment** âś… N/A â€” Superseded by CloudFront/CDN migration (2026-04-04). API responses now contain CDN URLs which do not expire, eliminating the TTL-alignment concern entirely.

- [x] **#18 `notifyFollowersOfNewEvent` and `notifyNearbyUsersOfNewEvent` perform unbounded `findMany` queries** âś… FIXED â€” Added `NOTIFICATION_FANOUT_LIMIT = 1000` constant and applied `take: NOTIFICATION_FANOUT_LIMIT + 1` to both `prisma.follow.findMany` (followers) and `prisma.user.findMany` (nearby users), slicing back to `NOTIFICATION_FANOUT_LIMIT` before processing. Using `LIMIT + 1` avoids false-positive warnings when the result count exactly equals the cap. Stable `orderBy` added to both queries for deterministic truncation. `@@index([followingId, createdAt, followerId])` added to `Follow` to support the ordered capped query efficiently. 98 tests pass.

- [x] **#19 Prisma query logging enabled in development includes full query text with parameters** âś… FIXED (PR #293) â€” Removed `'query'` from the dev log array in `PrismaService`. Prisma's query level interpolates actual parameter values into stdout; `'info'`, `'warn'`, `'error'` are parameter-free and retained. Unit test updated to assert `'query'` is never present.

### Low

- [x] **#20 No `generateStaticParams` for event or group detail pages** âś… FIXED â€” Added `export const revalidate = 60` to both event and group detail pages. `generateStaticParams` was not used because event/group IDs are user-generated and not known at build time. ISR caches the rendered HTML for 60 seconds; `fetchApi` already uses the same TTL at the fetch level. Client components refresh data on hydration via React Query.

## Database & Prisma

### High

- [x] **#21 TOCTOU race condition in capacity enforcement allows overbooking** âś… FIXED (PR #254) â€” All three capacity guard sites now wrap count + write in `$transaction` with `SELECT ... FOR UPDATE` on the Event row. In `rsvps.service.ts:create()` and `update()`, the count and RSVP write are inside a single transaction with the row lock; uncapped events bypass the transaction entirely. In `events.service.ts:updateEventRsvp()`, capacity is now read inside the transaction via `$queryRaw` on the locked row, eliminating the prior stale-read window where capacity was fetched before the lock. In `invitations.service.ts:respond()`, the `FOR UPDATE` lock is acquired before the count inside the existing transaction. 4 new unit tests verify atomicity across all three paths. Found in: `apps/api/src/rsvps/rsvps.service.ts`, `apps/api/src/events/events.service.ts`, `apps/api/src/invitations/invitations.service.ts`

### Medium

- [x] **#22 `Float` type used for geospatial coordinates instead of a precision-appropriate type** âś… FIXED (PR #273) â€” Added `@db.DoublePrecision` to `homeLatitude`/`homeLongitude` (User) and `latitude`/`longitude` (Event, Group). PostgreSQL already stores `Float` as `double precision`, so the migration is a no-op DDL change. Annotation now explicit and aligned with Prisma best practices.

- [x] **#23 No composite index on `(status, startDate)` for the primary events query** âś… FIXED (PR #296) â€” Added `@@index([status, startDate])` to the `Event` model. PostgreSQL can now satisfy `WHERE status = 'PUBLISHED' ORDER BY startDate` with a single composite index scan. Standalone `@@index([status])` retained to avoid query-plan regressions on status-only lookups. Migration `20260408000000_add_event_status_startdate_composite_index` included.

- [x] **#24 Orphaned `SentReminder_sentAt_idx` index on `SentReminder.sentAt`** âś… FIXED (PR #297) â€” Confirmed that `@@unique([eventId, userId])` fully covers the only query pattern (`WHERE eventId = ? AND userId IN (...)`). Removed `@@index([sentAt])` which was orphaned â€” no query filters or orders by `sentAt`. Migration `20260408010000_drop_sent_reminder_sent_at_index` drops the index.

- [x] **#25 `Notification.invitationId` field is not a foreign key, has no index, and can reference deleted invitations** âś… FIXED (PR #298) â€” Added `@@index([invitationId])` to `Notification`. No cascade added by design: notifications are an immutable historical log; `invitationId` is preserved after invitation deletion for audit purposes. Migration `20260408020000_add_notification_invitationid_index` included.

### Low

- [x] **#26 `Category` model has no `updatedAt` field** âś… FIXED â€” Added `updatedAt DateTime @updatedAt @db.Timestamptz` to the `Category` model. Migration `20260410000000_add_category_updatedat` adds the column with a temporary `DEFAULT now()` to backfill existing rows, then drops the default so it does not remain as a persistent DB-level default. All 13 category tests pass; TypeScript and lint clean.
